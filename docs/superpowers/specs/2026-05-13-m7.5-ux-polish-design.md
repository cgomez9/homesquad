## 1. Scope and exit criteria

### 1.1 Goal

Establish a design system foundation and apply it visibly to the auth flow, ship the signup password UX polish, and add Spanish localization with device-default detection. The other ~15 app screens stay on their current inline styles until touched in a future milestone.

### 1.2 In scope

1. **Design system foundation.** A single `mobile/src/theme.ts` file exporting `colors`, `spacing`, `radii`, `typography` tokens. Plain TypeScript objects — no styled-components, no Restyle, no provider/context. Refactor shared components (`Button`, `TextField`, `DeleteAccountModal`, `SocialAuthRow`) to consume tokens.
2. **Nunito font** loaded via `@expo-google-fonts/nunito`. Weights: 400 Regular, 600 SemiBold, 700 Bold. Splash screen stays visible until fonts are ready.
3. **Ocean palette** applied across the auth flow: teal primary, coral accent, sandy neutrals, seafoam success, sunset warning, deep coral error.
4. **Auth flow refactor** — five screens: `login.tsx`, `signup.tsx`, `reset.tsx`, `create-family.tsx`, `add-kid.tsx`.
5. **`PasswordField` component** — wraps `TextField` with an eye-icon toggle (Ionicons `eye` / `eye-off`) for password visibility. Used in `signup.tsx` and `login.tsx`.
6. **`PasswordStrengthMeter` component** — zxcvbn-based, five-segment colored bar + locale-aware label. Shown only on `signup.tsx`. Blocks Sign Up button when `length < 8 OR score < 2`.
7. **Spanish localization** via `i18next` + `react-i18next` + `expo-localization`. Two translation files (`en.json`, `es.json`). Device-default detection on first launch, with `AsyncStorage`-persisted manual override.
8. **Settings → Language section** with a picker modal (English / Español / System default).

### 1.3 Out of scope (deferred to M8+)

- Visual refactor of non-auth screens (kid home, chores list, rewards list, approvals, parent home, badges, profile picker, etc.)
- Dark mode (`userInterfaceStyle: 'light'` stays in `app.json`)
- Additional languages (Portuguese, French, etc.)
- RTL layout support
- Pluralization rules beyond basic `count` interpolation
- Custom font variants (no italic, no other weights)
- Component variants beyond strictly needed (no `Button size="large"`, no per-component color schemes)
- M6 placeholder sound asset replacement

### 1.4 Exit criteria

1. Five auth screens render in Nunito with Ocean palette on iOS dev build and Android dev build.
2. Same five screens render correctly in both English (device locale `en-*`) and Spanish (device locale `es-*`).
3. Signup screen shows a color-coded zxcvbn strength meter that updates as the user types; Sign Up button disabled until `length >= 8 AND score >= 2` AND confirm field matches.
4. Password fields on signup + login have a working eye-toggle icon that switches between masked and visible text.
5. Settings → Language allows manual switching between English / Español / System default with no app restart.
6. App relaunch preserves the manually selected language (or reverts to device locale when "System default" is selected).
7. All existing Jest tests still pass. New tests added for `PasswordStrengthMeter.isAcceptable()`, `PasswordField` eye toggle, `i18n.setLanguage`, and translation-parity.
8. `npx tsc --noEmit` clean.

### 1.5 Branch and ship strategy

M7.5 piggybacks on the in-flight `m7-prelaunch` branch (not main). M7 + M7.5 ship together when both are complete and external dependencies (Apple Dev enrollment, Google OAuth setup) clear.

```ts
export const colors = {

  border: '#D6E5E3',
  success: '#34D399',
  warning: '#F97316',
  error: '#E11D48',
  strengthVeryWeak: '#E11D48',
  strengthWeak: '#F97316',
  strengthFair: '#FACC15',
  strengthStrong: '#34D399',
  strengthVeryStrong: '#0EA5A4',
};

export const spacing = { xs: 4, sm: 8, md: 12, lg: 16, xl: 24, xxl: 32 };

export const radii = { sm: 8, md: 12, lg: 14, pill: 999 };

export const typography = {
  fontFamily: 'Nunito_400Regular',
  fontFamilySemi: 'Nunito_600SemiBold',

```

No theme provider, no context, no hooks. Components write `import { colors, spacing } from '../theme'` and use the values directly in `StyleSheet.create`.

### 2.2 Font loading (`mobile/app/_layout.tsx`)

Add `@expo-google-fonts/nunito` to dependencies. Load fonts at root layout and gate the rendered tree:

```tsx
import { useFonts, Nunito_400Regular, Nunito_600SemiBold, Nunito_700Bold } from '@expo-google-fonts/nunito';

const [fontsLoaded] = useFonts({ Nunito_400Regular, Nunito_600SemiBold, Nunito_700Bold });
if (!fontsLoaded) return null;
```

Splash screen stays visible until fonts are ready — no flash of unstyled text.

### 2.3 i18n setup (`mobile/src/i18n/index.ts`)

```ts
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import * as Localization from 'expo-localization';
import AsyncStorage from '@react-native-async-storage/async-storage';
import en from './locales/en.json';
import es from './locales/es.json';

const LANG_STORAGE_KEY = 'shores_lang_pref';

export async function initI18n() {
  const stored = await AsyncStorage.getItem(LANG_STORAGE_KEY);
  const deviceLocale = Localization.getLocales()[0]?.languageCode ?? 'en';
  const lang = stored ?? (deviceLocale === 'es' ? 'es' : 'en');

  await i18n.use(initReactI18next).init({
    resources: { en: { translation: en }, es: { translation: es } },
    lng: lang,
    fallbackLng: 'en',
    interpolation: { escapeValue: false },
  });
}

export async function setLanguage(lang: 'en' | 'es' | 'system') {
  if (lang === 'system') {
    await AsyncStorage.removeItem(LANG_STORAGE_KEY);
    const deviceLocale = Localization.getLocales()[0]?.languageCode ?? 'en';
    await i18n.changeLanguage(deviceLocale === 'es' ? 'es' : 'en');
  } else {
    await AsyncStorage.setItem(LANG_STORAGE_KEY, lang);
    await i18n.changeLanguage(lang);
  }
}

export function getCurrentLanguagePref(): Promise<'en' | 'es' | 'system'> {
  return AsyncStorage.getItem(LANG_STORAGE_KEY).then((v) => {
    if (v === 'en' || v === 'es') return v;
    return 'system';
  });
}
```

`initI18n()` is awaited in `_layout.tsx` alongside `useFonts`. Components use `const { t } = useTranslation()` and call `t('auth.login.title')`.

### 2.4 Translation files

Two files at `mobile/src/i18n/locales/{en,es}.json`. Only the in-scope screens have keys. The full payload appears in Section 4.

### 2.5 `PasswordField` component (`mobile/src/components/PasswordField.tsx`)

Wraps `TextField` with an eye-icon toggle. Props:

```ts
type Props = TextInputProps & {
  label: string;
  error?: string;
  showStrength?: boolean;
};
```

Internally:
- `useState<boolean>(false)` for `visible`.
- Passes `secureTextEntry={!visible}` to the underlying `TextField`.
- Renders an Ionicons `eye` (when masked) / `eye-off` (when visible) icon absolutely positioned at right: 12, with hitSlop for an easy tap target.
- When `showStrength` is true, renders `<PasswordStrengthMeter value={value} />` below.

### 2.6 `PasswordStrengthMeter` (`mobile/src/components/PasswordStrengthMeter.tsx`)

Five-segment colored bar + locale-aware label. Implementation:

- Imports `zxcvbn` synchronously at module top. The library is ~400KB unpacked / ~150KB gzipped. The synchronous import lets `isAcceptable()` (the signup button gate) be a pure synchronous function.
- On every render: `const result = zxcvbn(value)` returning `{ score: 0|1|2|3|4 }`.
- Five horizontal segments (each `flex: 1`, height 4, gap 4 between). Segments at or below `score` get the score color; segments above get `colors.border`.
- Label below uses `colors.textMuted` `small` font, content from `t('auth.passwordStrength.<level>')`.

Score → color mapping:
- 0 (`tooShort` if length < 8, else `veryWeak`) → `strengthVeryWeak` (`#E11D48`)
- 1 (`weak`) → `strengthWeak` (`#F97316`)
- 2 (`fair`) → `strengthFair` (`#FACC15`)
- 3 (`strong`) → `strengthStrong` (`#34D399`)
- 4 (`veryStrong`) → `strengthVeryStrong` (`#0EA5A4`)

Static helper:
```ts
export function isAcceptable(value: string): boolean {
  if (value.length < 8) return false;
  return zxcvbn(value).score >= 2;
}
```

Called by signup to gate the Sign Up button.

### 2.7 Settings → Language row

Appended to `mobile/app/(app)/parent/settings.tsx`, above the existing Account section:


## 3. Auth flow screen rework specifics

The five auth screens get the same visual treatment.

**Spacing:** `spacing.xl` (24) between fields, `spacing.xxl` (32) around the title block.

**Typography:**
- Title: `Nunito_700Bold`, size `typography.h1` (28).
- Field label: `Nunito_600SemiBold`, size `typography.small` (13), color `colors.text`.
- Body text: `Nunito_400Regular`, size `typography.body` (15).

**Input styling:**
- Background: `colors.surface`.
- Border: `colors.border` 1px.
- Radius: `radii.lg` (14).
- Padding: 14 vertical, 12 horizontal.
- Focused state: border becomes `colors.primary`.
- Error state: border becomes `colors.error`; helper text below in `colors.error`.

**Primary button (`<Button variant="primary">`):**
- Background: `colors.primary`.
- Pressed: `colors.primaryDark`.
- Disabled: `opacity: 0.5`.
- Radius: `radii.lg` (14).
- Height: 52.
- Label: `Nunito_700Bold` `typography.body` (15), color `#FFFFFF`.

**Secondary button (`<Button variant="secondary">`):**
- Background: `transparent`.
- Border: `colors.primary` 1.5px.
- Label: `colors.primary`.
- Same radius and height as primary.

**Link text:**
- Color: `colors.primary`.
- Font: `Nunito_600SemiBold`.
- No underline (RN convention).
- Padding: `spacing.sm` vertical, generous touch target.

**Signup-specific:**
- Password field uses `<PasswordField showStrength={true}>`.
- Confirm-password field uses `<PasswordField>` (no strength meter on confirm).
- Sign Up button stays disabled with `opacity: 0.5` until:
  - `PasswordStrengthMeter.isAcceptable(password)` returns true, AND
  - `password === confirm`, AND
  - `email.trim().length > 0`.

**Login:** same input/button treatment, no strength meter. Password field uses `<PasswordField>`.

**Reset, create-family, add-kid:** input/button treatment matches. Reset has no password field. Create-family and add-kid keep their existing widgets (avatar picker, family-name field, etc.) — the visual update is just spacing/colors/font on the shared chrome.

## 5. Testing and edge cases

### 5.1 Automated test coverage

| Target | Test | Coverage |
|---|---|---|
| `PasswordStrengthMeter.isAcceptable()` | Jest | `''` → false, `'short1!'` → false (under 8), `'correcthorsebattery'` → true (score≥2), `'password123'` → false (common pattern), `'a'.repeat(100)` → false (low entropy via repetition) |
| `PasswordField` | Jest + RTL | `secureTextEntry` true by default; tapping eye flips it; second tap flips back |
| `i18n.setLanguage` | Jest | `setLanguage('es')` persists to AsyncStorage + calls `changeLanguage`; `setLanguage('system')` clears storage + reverts to device locale (mocked) |
| Translation parity | Jest | Depth-first walk of `en.json` asserts every key has a corresponding key in `es.json`. Catches missing translations the moment they're added |
| Existing tests | unchanged | All current Jest suites must continue to pass after Button/TextField refactor |

### 5.2 Manual acceptance

Run on Android emulator first; iOS dev build once Apple Dev clears.

1. Fresh launch with device locale `en-US` → app loads in English; all five auth screens visible in Ocean palette + Nunito.
2. Change device locale to `es-MX` → relaunch → app loads in Spanish.
3. Settings → Language → switch to English manually → screen updates without restart, persists across relaunch.
4. Settings → Language → System default → app reverts to device locale.
5. Signup: type `password` → meter shows red "Muy débil" → Sign Up disabled. Type `correcthorsebattery` → meter green "Fuerte" → Sign Up enabled.
6. Tap eye icon on password field → text reveals. Tap again → masks.
7. Submit signup → success → user lands in app.

### 5.3 Edge cases

- **Font load failure** — `useFonts` gates splash. If font load errors, RN falls back silently to system font. Mitigation: 5-second timeout after which we proceed even if fonts aren't ready (logs to console).
- **AsyncStorage failure on `setLanguage`** — wrap in try/catch; still call `i18n.changeLanguage` so the user sees the change for this session.
- **Device locale variants** — only the language code (`es`, not `es-MX`) is checked. All `es-*` variants map to our `es` translations. Latin American Spanish wording chosen for accessibility ("tú" not "vos", "computadora" not "ordenador" — neutral LATAM).
- **zxcvbn bundle size** — adds ~400KB unpacked / ~150KB gzipped to the JS bundle. Accepted as a synchronous import for `isAcceptable()` simplicity. Net app size impact is small relative to the existing native modules.
- **Password field with autofill** — iOS/Android password managers may fill via `autoComplete="new-password"`. The strength meter still updates because `onChangeText` fires.
- **Non-Latin password characters** — zxcvbn handles unicode reasonably; we don't restrict input.

## 7. Decision log

| # | Decision | Alternative considered | Why |

| 1 | Foundation + auth flow showcase (Option B in brainstorm Q1) | Foundation only / full refactor | Visible UX win where users actually touch the app; rest of screens follow incrementally |

| 5 | NIST-aligned hybrid password rules (Option C in Q5) | Strict regex / soft meter | Modern, helpful, prevents obviously-weak passwords |
| 6 | Medium-depth token system | Lightweight / Restyle library | Right balance of structure vs YAGNI |
